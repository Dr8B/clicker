# AutoClicker — автоматический кликер на Go

Простой автокликер с возможностью настройки профилей через конфигурационный файл. Активируется удержанием правого `Ctrl`.

## Особенности

* Активация по удержанию правого `Ctrl` (правый Ctrl).
* Поддержка нескольких профилей действий.
* Настройка кликов мышью и нажатий клавиш.
* Низкая нагрузка на CPU за счёт паузы между проверками состояния клавиши.
* Конфигурация через YAML‑файл.
* Удобная сборка и управление проектом через Taskfile.

## Установка и настройка

### Требования

* Go 1.16+
* ОС Windows (из‑за использования `user32.dll`)
* Установленный [Task](https://taskfile.dev/) (альтернатива Make)
* Установленные зависимости:

```bash
go get github.com/go-vgo/robotgo
go get github.com/goccy/go-yaml
go get golang.org/x/sys/windows
```

## Сборка проекта с использованием Taskfile

1. Клонируйте репозиторий:

```bash
git clone <url-репозитория>
cd <папка-проекта>
```

2. Используйте команды Task для управления проектом:

### Доступные команды Task

| Команда | Описание | Пример использования |
|-------|---------|------------------|
| `task` | Выводит приветственное сообщение (команда по умолчанию) | `task` |
| `task clean` | Очищает собранные бинарники | `task clean` |
| `task build` | Собирает Go‑проект в бинарный файл | `task build` |

### Параметры сборки (можно переопределить)

В `Taskfile.yml` определены следующие переменные:

* `BINARY_NAME` — имя выходного бинарника (по умолчанию: `clicker`).
* `GOOS` — целевая операционная система (по умолчанию: `windows`).
* `GOARCH` — целевая архитектура (по умолчанию: `amd64`).
* `CGO_ENABLED` — включение CGO (по умолчанию: `1`).

**Примеры использования:**

Сборка с параметрами по умолчанию:
```bash
task build
```

Сборка для другой архитектуры:
```bash
task build -- GOARCH=386
```

Переопределение имени бинарника:
```bash
task build -- BINARY_NAME=myautoclicker
```

## Конфигурация

Настройте поведение кликера в файле `config.yaml`.

### Пример `config.yaml`

```yaml
profile: "default"
profiles:
  default:
    clicks:
      - "left"   # левый клик
      - "right"  # правый клик
    keys:
      - "space"  # пробел
      - "a"      # клавиша A
  game_profile:
    clicks:
      - "left"
    keys:
      - "1"
      - "2"
```

### Параметры конфигурации

* `profile` — имя активного профиля (обязательно).
* `profiles` — список доступных профилей.
* `clicks` — массив типов кликов (`"left"`, `"right"`, `"middle"`).
* `keys` — массив клавиш для нажатия (используются названия из `robotgo`).

## Использование

1. Настройте `config.yaml` под свои нужды.
2. Соберите проект:

```bash
task build
```

3. Запустите программу:

```bash
./clicker.exe
```
(или с другим именем, если вы его переопределили)

4. Удерживайте правый `Ctrl`, чтобы активировать автокликер.
5. Отпустите правый `Ctrl`, чтобы остановить автокликер.
6. Нажмите `Ctrl+C` в консоли, чтобы полностью завершить работу программы.

## Логирование

Программа выводит сообщения в консоль:

* «Автокликер запущен. Зажмите правый Ctrl для активации. Ctrl+C — выход.» — при старте.
* «Активирован автокликер (Ctrl зажат)» — при активации.
* «Автокликер остановлен (Ctrl отпущен)» — при деактивации.

## Примечания

* Программа использует `GetAsyncKeyState` из `user32.dll` для отслеживания состояния клавиши `Ctrl`.
* Частота действий ограничена паузой в 100 мс между проверками (для снижения нагрузки на CPU).
* Для кликов и нажатий клавиш используется библиотека `robotgo`.
* Убедитесь, что указанный в `config.yaml` профиль существует и содержит хотя бы одно действие (`clicks` или `keys`).

## Ограничения

* Работает только на Windows.
* Требует прав пользователя для имитации ввода.
* Не поддерживает настройку координат мыши — клики выполняются в текущей позиции курсора.

## Устранение проблем

**Ошибка чтения файла `config.yaml`**

Проверьте:
* существование файла `config.yaml` в рабочей директории;
* права доступа к файлу;
* корректность синтаксиса YAML.

**Профиль не найден**

Убедитесь, что:
* параметр `profile` в `config.yaml` соответствует имени одного из профилей в `profiles`;
* профиль содержит хотя бы один элемент в `clicks` или `keys`.

**Кликер не реагирует на `Ctrl`**

* Проверьте, что удерживается именно правый `Ctrl` (`VK_RCONTROL`).
* Убедитесь, что другие приложения не перехватывают нажатие клавиши.

**Проблемы со сборкой через Task**

* Убедитесь, что Task установлен и доступен в PATH.
* Проверьте версию Go и совместимость с зависимостями.
* При необходимости выполните `go mod tidy` для обновления зависимостей.
